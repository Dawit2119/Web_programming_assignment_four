Does it matter using relative units in CSS?

Units in CSS

  • There are two measurement units in CSS those are:

    • Relative measurements and
    • Absolute measurements. in this section only Relative measurements and the matter of whether using or not them will be discussed.

  • Relative measurements.

      • Relative measurement units are based on or relative to other property of elements. Basically there three classes of Relative measurements. Those are:
    • based on the view port property(vh,vw,vmax and vmin) these are depend on the broswer’s width or height property or simply the viewport of the browser dimensions. All divide the viewport into 100 by 100 grid.
      • vw (viewport width) this relative to the viewport width. eg 50vm means half of the browser viewport width dimension.
      • vh (viewport-width) this relative to the viewport height. eg 50vh means half of the browser viewport height dimension.
      • vmax (viewport maximum) this is relative to the maximum dimension of the browser’s viewport either its width or height.
      • vmin (viewport minimum) this is relative to the minimum dimension of the browser’s viewport either its width or height.
    • based on the currently declared font-attributes (em,rem,ex,ch), this measurement units are relative to the font-attributes to the root ,parent element or character sizes.
      • em – this measurement unit is relative to the font-size of the parent element. The values are cascaded to all the children if exists, this makes unexpected and unintended result for the author.
      • rem – this measurement unit is relative to the font-size the root element or simply which is the HTML
      • ex – this is relative to x-height of character or the current font-family.
      • ch – this measurement unit is relative to the 0-height of characters
    • based on the parent dimensions
      • percentage(%) Relative to the parent width or height.

The main reason of using relative measurement units in website devlopment.

my answer for the question in the title at the beggining is yes, because using relative units are important:

  • to maintain responsiveness and flexibility of the site. Because relative units allow the element’s size ,text-size and spacing to grow or shrink proportionally to the font-size of the parent or root elements.
  • We can not maintain responsiveness of the page using absolute unit measurements,such as pixels, because they stay the same as declared irrespective of the device the page will be displayed.
  • If the default font-size has given to the browser, it is not mandatory to zoom in the browser to be able to read a website,because the content of a website with relative units will shrink or grow with your preferred font size.
  • General speaking, to make the website more responsive, using relative units such as em and rem for text elements is very necessary.

Default browser font-size

We said 16px is default browser width so what does it mean when we say 200px ? How does the browser interpret that ?

default browser’s font-size is 16px for most browsers. That means the text under the page or all of the entire html element is 16px unless any font-attribute is introduced. Even though this is the default browsers font-size it can be changed based on the users preferences.for example partially sighted ones are making this to little bigger. When making the default font-size to 200px (introducing css i.e. html{font-size:200px) making the text size to grow to very large percentage. i.e. 16px is to 100% and 200px means (200*100)/16 = 1250%

The specific changes in CSS3

here are the most awesome css3 features that most modern browsers can handle

  • CSS Animations and Transitions: these are available in all major browsers. There are two ways to create CSS animations:
    1. using the transition declaration and this is triggered by mouse or hover action or using javascript
    2. using the code @keyframe rule, and this is not depend on the user actions
  • Calculating values with calc(), it allows to do simple arithmetic calculations to specify value for attributes of elements.
  • Advanced Selectors, CSS3 introduced a number of powerful selectors that can make a layouts cleaner.
  • Generated content and counters, ::before and ::after pseudo elements, this elements use less HTML to achieve the same layouts. CSS3 also gives pseudo elements to counters which can be incremented width CSS rule.
  • Webfonts, CSS3 introduced many font-services like google Fonts , typekit and fontawesome for icons.
  • Box Sizing , it makes life easier by removing extra padding and margin values added to the element’s size.
  • Border-images, this property allows to display custom designed borders around elements.
  • Media Queries. This property helps the website to be responsive and which adapt to the type of devices. It is simple to use all the need is to enclose CSS styles in a block guarded bu a code @media rule.
  • Multiple backgrounds, using this property the designers can achieve very interesting effects.
  • CSS columns, CSS3 provides a way to display columns using the property columns rule.
  • Gradients ,Gradients give web designers the power to create smooth transitions between colors without having to resort to images.
  • CSS 3D Transforms, css3 offers powerful features to designers and developers that can attract the users.
